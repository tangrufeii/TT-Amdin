# 项目说明

## 定位
TT Admin 面向插件化与多业务线扩展，支持前后端插件动态加载，适合企业级后台统一治理。

## 架构
- 后端：DDD 分层 + 插件引擎（域层、应用层、基础设施层、接口层）。
- 前端：模块化路由 + 插件 UI 运行时加载。
- 数据：权限与菜单统一管理，插件与宿主共用权限模型。

## 权限模型
RBAC 为核心，菜单与按钮权限分离，按钮级权限建议统一命名规范。

## 前端技术栈
Vite + Vue3 + Naive UI + UnoCSS，支持主题预设与统一样式。

## 后端技术栈
Spring Boot 3 + JDK21 + MyBatis-Plus，插件安装与生命周期托管。

## 插件 UI 接入
- 开发环境：`/plugin-dev` 访问插件 UI。
- 生产环境：`/api/plugin-static` 访问插件 UI。
- 插件 UI 统一使用 `@tt/plugin-sdk` 发起请求，避免重复封装。

## UI 规范
- 表头统一使用 `TableHeaderOperation`，按钮权限通过 `add-auth / delete-auth / export-auth` 控制。
- 表单优先使用抽屉表单，页脚按钮对齐宿主规范（取消/确认）。
- 列表页建议使用 `h-full-hidden`，避免滚动容器高度不一致。

## 主题与样式
主题覆盖与组件间距已对齐宿主风格，插件开发请避免自定义全局样式。

## 落地建议
新业务优先做插件，保持宿主轻量，便于独立迭代与上线。
